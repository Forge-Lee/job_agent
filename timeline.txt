Day 1：项目初始化

目标：

创建 GitHub repo
写 README
写 workflow_design.md
准备 sample_jd.txt
准备 candidate_profile.json
配置 .gitignore / requirements.txt / Dockerfile

这一天不需要写太多功能代码。

Day 2：实现 JD Parser

目标：

输入 sample_jd.txt
输出 parsed_jd.json

先保证结构化提取稳定。

Day 3：实现 Profile Matcher

目标：

读取 parsed_jd.json
读取 candidate_profile.json
输出 match_result.json

先写 rule-based score，再用 LLM 补充自然语言解释。

Day 4：实现 Material Generator

目标：

生成 match_report.md
生成 cover_letter.md
生成 linkedin_message.md

这一天开始项目就能展示了。

Day 5：实现 Tracker + CLI

目标：

python -m src.main --jd data/sample_jd.txt --profile data/candidate_profile.json

输出文件保存到 outputs/。

Day 6：Docker 化

目标：

docker compose up --build

让别人 clone repo 后可以直接跑。

Day 7：完善 README + sample output

目标：

README 里展示 workflow、input、output、how to run、future work

这一步非常重要。GitHub 项目是否有说服力，很大程度取决于 README。
